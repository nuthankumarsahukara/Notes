 What is Virtual DOM (VDOM)?
Virtual DOM is a lightweight in-memory representation of the real DOM.
It is a JavaScript object that React uses to optimize and efficiently update the UI.

How It Works in React:
Initial Render:
	1.React creates a VDOM based on the JSX code.
	2.It converts this VDOM into real DOM and mounts it.

Re-render (on state/props change):
	1.React creates a new VDOM.
	2.Compares it with the previous VDOM (using diffing algorithm).
	3.Only the differences (minimal set of changes) are updated in the 			real DOM.

Advantages of VDOM:
Performance Boost: Reduces direct DOM manipulations.
Efficient Re-rendering: Updates only the changed parts of the DOM.
Cross-platform: Can be used for both web and mobile (via React Native).
Improved User Experience: Faster updates lead to smoother interfaces.

Key Concepts:
Diffing: Comparing old and new VDOM trees to detect changes.
Reconciliation: Updating the real DOM with the changes.
Render: JSX → React elements (VDOM)
ReactDOM.render(): VDOM → Real DOM

VDOM Example:
function Counter() {
  const [count, setCount] = React.useState(0);
  return <button onClick={() => setCount(count + 1)}>Count: {count}</button>;
}

Initial state: VDOM → <button>Count: 0</button>
On click: state updates → new VDOM → <button>Count: 1</button>
React compares old vs new VDOM → only updates text node in real DOM

import { useState } from "react"

function App(){
    let [counter,setCounter]=useState(0);
    let items=[
        "Nuthan","Sai","Kumar","Vinod","Ganesh","Shiva"
    ]
    return(
        <>
        <h1>Count : {counter}</h1>
        <button onClick={
            ()=>{setCounter(counter++)}
        }>Increment</button>
        <ul>
        {
            items.map((a,index)=>(
                <li key={index}>{a}</li>
            ))
        }
        </ul>
        </>

    )
}

export default App;

DOM Example:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <div id="counter">
    <h1>count : 0</h1>
    <button id="increment-btn">Increment</button>
    <ul id="list"></ul>	
    </div>
    <script>
        let count =0;
        const counterE1=document.querySelector('#counter');
        const incrementBtnE1=document.querySelector('#increment-btn');
        const listE1 =document.querySelector('#list');

        incrementBtnE1.addEventListener('click' , () =>{
            count++;
            render();
        });

        function render(){
            const countE1=counterE1.querySelector('h1');
            countE1.textContent=`count :${count}`;
            listE1.innerHTML=' ';
            for(let i=1;i<=10;i++){
                const itemE1=document.createElement('li');
                itemE1.textContent=`Items ${i}`;
                listE1.appendChild(itemE1);
            }
        }
        render();
    </script>
</body>
</html>

In Simple Words:
DOM: Real structure shown in the browser. It's heavy and slow if manipulated too much.
VDOM: A smart copy of the real DOM that React uses to figure out the most efficient way to update the UI.

Visual Analogy:
DOM = Final printed document
VDOM = Editable draft version that helps React make the best update decision
