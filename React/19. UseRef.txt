1. What is useRef?
useRef is a hook in React used to persist a mutable reference that does not cause a re-render when updated.

Commonly used to access DOM elements or store mutable values.

useRef can store values that persist across renders without causing a re-render.

ref={...} is used to attach a ref object to a DOM element.

2. Syntax
import { useRef } from 'react';

const myRef = useRef(initialValue);

3. Use Cases
a) Accessing DOM elements

function InputFocus() {
  const inputRef = useRef(null);

  const focusInput = () => {
    inputRef.current.focus();
  };

  return (
    <>
      <input ref={inputRef} type="text" />
      <button onClick={focusInput}>Focus Input</button>
    </>
  );
}

Avoiding Re-renders on Value Changes
useRef changes do not trigger re-renders, unlike useState.

Difference: useRef vs createRef
createRef: Used in class components; creates a new ref every render.

useRef: Used in function components; persists the same ref throughout the componentâ€™s lifecycle.

When to Use useRef :-
Accessing or manipulating DOM nodes.
Storing values across renders without triggering re-renders.
Keeping track of previous props or state values.

Ex:-
import {useRef} from 'react';
function App(){
  const count=useRef(0);
  const text=useRef(null);
  return(
    <>
        <h1 ref={text}>Count : 0 </h1>
        <button onClick={
          ()=>{
            count.current++;
            text.current.innerText= "Count :"+count.current;
          }
        }>Count Update</button>
    </>
  )
}

export default App;
