Date/Time Functions :-

ADD_MONTHS function returns a date with a specified number of months added.
Syntax: ADD_MONTHS (datel, number_months)

CURRENT_DATE function returns the current date in the time zone of the current SQL session as set by the ALTER SESSION command.
Syntax: CURRENT_DATE

CURRENT_TIMESTAMP function returns the current date and time in the time zone of the current SQL session as set by the ALTER SESSION command. It returns a TIMESTAMP WITH TIME ZONE value.
Syntax: CURRENT_TIMESTAMP

DBTIMEZONE function returns the database time zone as a time zone offset (in the following format: '[+/-]TZH:TZM') or a time zone region name.
Syntax: DBTIMEZONE

EXTRACT function extracts a value from a date or interval value.
Syntax :EXTRACT (
{YEAR | MONTH | DAY | HOUR | MINUTE | SECOND} | { TIMEZONE_HOUR | TIMEZONE_MINUTE }
*{ TIMEZONE_REGION | TIMEZONE_ABBR} FROM { date_value | interval_value })

Note:-
1.You can only extract YEAR, MONTH, and DAY from a DATE.
2.You can only extract TIMEZONE_HOUR and TIMEZONE_MINUTE from a timestamp with a time zone datatype.

Returns:-
1.The EXTRACT function returns a numeric value when the following parameters are provided: YEAR, MONTH, DAY, HOUR, MINUTE, SECOND, TIMEZONE_HOUR, TIMEZONE_MINUTE, TIMEZONE_REGION, TIMEZONE_MINUTE.
2.The EXTRACT function returns a VARCHAR2 when TIMEZONE REGION or TIMEZONE ABBR parameters are provided (because the time zone name or abbreviation information is returned).

SELECT EXTRACT(Month FROM DATE '1998-03-07') from dual;
 SELECT EXTRACT(TIMEZONE_minute FROM TIMESTAMP '1999-01-01 10:51:59 -05:30') from dual;

LAST_DAY function returns the last day of the month based on a date value.
Syntax: LAST_DAY(date)

LOCALTIMESTAMP function returns the current date and time in the time zone of the current SQL session as set by the ALTER SESSION command. It returns a TIMESTAMP value.
Syntax: LOCALTIMESTAMP

Note:-
1.A similar function to the LOCALTIMESTAMP function is the CURRENT TIMESTAMP function. 2.The difference between these two functions is that the LOCALTIMESTAMP function returns a TIMESTAMP value while the CURRENT_TIMESTAMP function returns a TIMESTAMP WITH TIME ZONE value.

MONTHS_BETWEEN function returns the number of months between date 1 and date2.
Syntax: MONTHS_BETWEEN( date1, date2)

NEW_TIME function converts a date from time zone1 to a date in time zone2. The NEW_TIME function returns a date value.
Syntax: NEW_TIME( date, zonel, zone2)

NEXT_DAY function returns the first weekday that is greater than a date.
Syntax: NEXT_DAY( date, weekday)
1-Sunday
7-Saturday

ROUND function returns a date rounded to a specific unit of measure.
Syntax: ROUND( date [, format] )

SESSIONTIMEZONE function returns the current session's time zone as a time zone offset (in the following format: '[+/-]TZH:TZM') or a time zone region name.
Syntax: SESSIONTIMEZONE

SYSDATE function returns the current system date and time on your local database.
Syntax: SYSDATE

SYSTIMESTAMP function returns the current system date and time (including fractional seconds and time zone) on your local database.
Syntax: SYSTIMESTAMP

TRUNC function returns a date truncated to a specific unit of measure. 
Syntax: TRUNC

TZ_OFFSET function returns the time zone offset of a value.
Syntax: TZ_OFFSET (timezone)
