Graphical user Interface:
GUI stands for Graphical User Interface, which is a way for users to interact with electronic devices using visual indicators and graphical icons. GUIs are often used instead of text-based user interfaces, which require users to type text commands or navigate using text.

In AWT (Abstract Window Toolkit), various layout managers are used to arrange components in a container in a specific way. Here's an overview of the layout managers you mentioned:

1.  Border Layout 
   The  BorderLayout  divides the container into five regions:  North ,  South ,  East ,  West , and  Center . Each region can hold one component, and the layout adjusts the size and position of components based on the container's size. Components in the  Center  region will expand to fill the available space, while others are resized according to their own preferred sizes.

Example: 
import java.awt.*;

class MyFrame2 extends Frame
{
	Button b1,b2,b3,b4,b5;
	public MyFrame2()
	{
		super("Welcome to AWT");
		b1=new Button("A");
		b2=new Button("B");
		b3=new Button("C");
		b4=new Button("D");
		b5=new Button("E");
		add(b1,BorderLayout.NORTH);
		add(b2,BorderLayout.SOUTH);
		add(b3,BorderLayout.EAST);
		add(b4,BorderLayout.WEST);
		add(b5,BorderLayout.CENTER);
		setSize(400,500);
		setVisible(true);
	}
	public static void main(String args[])
	{
		new MyFrame2();
		
	}
}

2.  Flow Layout 
   The  FlowLayout  manager arranges components in a left-to-right flow, like words on a page. When the container's width is filled, it wraps the components to the next line. It can be used with three alignment options:  left ,  center , and  right .

3.  Grid Layout 
   The  GridLayout  manager arranges components in a grid of rows and columns. All components have the same size, and the layout adjusts to fill the grid. You define the number of rows and columns when initializing the layout.
Example:
   import java.awt.*;
   public class GridLayoutExample {
       public static void main(String[] args) {
           Frame frame = new Frame("Grid Layout Example");
           frame.setLayout(new GridLayout(2, 3)); // 2 rows, 3 columns
           
           frame.add(new Button("Button 1"));
           frame.add(new Button("Button 2"));
           frame.add(new Button("Button 3"));
           frame.add(new Button("Button 4"));
           frame.add(new Button("Button 5"));
           frame.add(new Button("Button 6"));
           
           frame.setSize(400, 300);
           frame.setVisible(true);
       }
   }

  4.  GridBag Layout 
   The  GridBagLayout  is a more flexible and complex layout manager. It allows you to specify the row and column position of components in a grid, and components can span multiple rows or columns. You control their alignment, resizing behavior, and other properties by using  GridBagConstraints .

Example:
   import java.awt.*;
   public class GridBagLayoutExample {
       public static void main(String[] args) {
           Frame frame = new Frame("GridBag Layout Example");
           frame.setLayout(new GridBagLayout());
           GridBagConstraints gbc = new GridBagConstraints();
           
           gbc.gridx = 0;
           gbc.gridy = 0;
           frame.add(new Button("Button 1"), gbc);
           
           gbc.gridx = 1;
           gbc.gridy = 0;
           frame.add(new Button("Button 2"), gbc);
           
           gbc.gridx = 0;
           gbc.gridy = 1;
           frame.add(new Button("Button 3"), gbc);
           
           gbc.gridx = 1;
           gbc.gridy = 1;
           frame.add(new Button("Button 4"), gbc);
           
           frame.setSize(400, 300);
           frame.setVisible(true);
       }
   }
  5.  Card Layout 
   The  CardLayout  allows you to stack components like a deck of cards, where only one card is visible at a time. You can switch between different cards (components) by invoking methods like  next() ,  previous() , or  show() .

Example:   
   import java.awt.*;
   public class CardLayoutExample {
       public static void main(String[] args) {
           Frame frame = new Frame("Card Layout Example");
           CardLayout cardLayout = new CardLayout();
           Panel panel = new Panel();
           panel.setLayout(cardLayout);
           
           panel.add(new Button("Card 1"), "Card 1");
           panel.add(new Button("Card 2"), "Card 2");
           panel.add(new Button("Card 3"), "Card 3");
           
           frame.add(panel);
           
           // Show the second card
           cardLayout.show(panel, "Card 2");
           
           frame.setSize(400, 300);
           frame.setVisible(true);
       }
   }
Summary of Layout Managers:

1.  BorderLayout : Divides the container into five regions (North, South, East, West, Center).
2.  FlowLayout : Components are arranged in a line and wrap to the next line when space is insufficient.
3.  GridLayout : Components are arranged in a grid with equal-sized cells.
4.  GridBagLayout : Flexible grid layout allowing components to span multiple rows and columns, with precise control over positioning and resizing.
5.  CardLayout : Displays one component at a time, simulating a deck of cards, with easy navigation between them.
